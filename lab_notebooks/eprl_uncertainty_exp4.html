There is a three-way interaction between accuracy during priming, object value, and incremental reliability: Participants with greater accuracy during incremental priming choose fewer low-valued old objects when the incremental signal is reliable.¶

  • When the incremental signal is reliable, participants only choose high-valued episodes more frequently than low-valued episodes if they are aware that the incremental signal is reliable. This implies that participants are better at avoiding low-valued episodes if they are aware the incremental signal is reliable.

  • When the incremental signal is unreliable, participants will choose high-valued old objects more frequently than low-valued old objects regardless of whether they are aware that the incremental signal is unreliable.

Speculation about possible explanations¶

You will default to the reliable incremental signal unless you are very certain about the episode being valuable¶

This result implies that both the certainty of the incremental controller and the episodic controller play a role in arbitrating choice. Determining that an incremental signal is reliable may allow for more resources to determine whether an episode is 1) previously seen and 2) valuable. Perhaps this is because an easy heuristic can be employed: choose incrementally unless the episode is highly valuable (In this case, more valuable than the incremental signal). What about more continuous value? How much more valuable does an episode need to be in order to be remembered using this heuristic?

Having an unreliable incremental signal (either being unaware or realizing it is unreliable) may lead to less selectivity¶

Determining that an incremental signal is unreliable does not seem to change baseline behavior here, which is to choose episodes more frequently and less selectively. Thus, if you don't have a reliable incremental signal (e.g. you aren't aware of it or you are and you've determined it isn't reliable) then you are going to choose episodes less selectively. This could be interpreted several ways. Perhaps it is that you are more willing to risk an episodic choice since choosing incrementally is less safe. Another explanation could be that it is only worth retrieiving less rewarding episodes in order to avoid them if the reliability heuristic (mentioned above) is available.

People who are better at priming accuracy are also just better at the task/are trying harder¶

It is also possible that this is just an artifact of performance (and people will perform better when they are more certain), although including performance as a covariate does not change the findings.

New hypothesis: The degree of uncertainty in the incremental signal is used to determine how valuable an episode needs to be in order to guide behavior¶

This is interesting because we now have evidence that some participants are making choices considering the incremental stimulus first, rather than the objects, which is necessary for our hypothesis to be tested.

We can see at the group level that this strategy does NOT explain a majority of participants' choices.

Conclusions¶

  • During Training:

    • Participants show a preference for reliable > unreliable circles and high old > low old. This replicates training effects from previous experiments, demonstrating that people can learn the task.

  • During Testing:

    • Participants in general follow the pattern from the previous experiments, where they demonstrate a preference for high old > low old but not reliable > unreliable. However, if we split participants by their accuracy on circle priming, we see that those who correctly indicate circle identity do show the reliable > unreliable choice preference.

    • There is no interaction between value and reliability.

    • There is a three-way interaction between value, reliability, and accuracy during priming such that individuals who are more accurate choose fewer low-valued objects when the incremental signal is reliable.

    • Most participants follow either the choose circle or choosehighold strategy as before. We also see an indication that priming with circle identity influences choices with the emergence of a strategy where some participants choose mostly based on whether a circle is reliable or not.

    • Participants that choose more reliable > unreliable circles also choose more objects when choosing between an object and an unreliable circle, but this is not greater for old objects compared to new objects.

def loadData(dType=None,dataDir="/Users/jonathan/shohamylab/jspych-exps/data/exp4",fileType="stateData"):
    data, exclude_pids = [], [] 
    dataFiles = [d for d in os.listdir(dataDir) if dType in d]
    for f in dataFiles:
        subData = pd.read_csv(os.path.join(dataDir,f))
        if dType == "interaction":
            if len(subData[subData.event == 'blur']) >= exclude_blur:
                exclude_pids.append(f.split('_')[0])
            subData['subjectID'] = [f.split('_')[0]] * len(subData)
        elif dType == "experiment":
            bonus = float(f.split('_')[2].replace('bonus',''))
            subData['bonus'] = [bonus] * len(subData)
        data.append(subData)
    data = pd.concat(data,ignore_index=True)
    return data, exclude_pids

def get_only_acc_trials(pid=None):
    
    inData = data[(data.test_part == "preTest") &
                  (data.subjectID == pid) &
                  (data.trial_number <= 150)]
    preTestAcc = []
    for i_row, row in inData.iterrows():
        if row.deck_type == "orange":
            if row.choice == "Unrel":
                accuracy = 1
            else:
                accuracy = 0
        elif row.deck_type == "blue":
            if row.choice == "Rel":
                accuracy = 1
            else:
                accuracy = 0
        preTestAcc.append(accuracy)
    inData["accuracy"] = preTestAcc
    acc_trials = list(inData[inData.accuracy == 1].trial_number)
    return acc_trials

def get_preTest_accuracy():

    preTestData = data[(data.test_part == "preTest") &
                       (data.trial_number <= 150)]
    preTestAcc = []
    for i_row, row in preTestData.iterrows():
        if row.deck_type == "orange":
            if row.choice == "Unrel":
                accuracy = 1
            else:
                accuracy = 0
        elif row.deck_type == "blue":
            if row.choice == "Rel":
                accuracy = 1
            else:
                accuracy = 0
        preTestAcc.append(accuracy)
    preTestData["accuracy"] = preTestAcc

    preTestAcc = {'pid':[],'accuracy':[]}
    for pid in preTestData.subjectID.unique():
        pidData = preTestData[preTestData.subjectID == pid]
        preTestAcc['pid'].append(pid)
        preTestAcc['accuracy'].append(pidData["accuracy"].mean())

def get_blurs(x):
    if str(x) in n_blurs.subjectID.unique():
        pid_blurs = n_blurs[n_blurs.subjectID == str(x)].n_blurs.iloc[0]
        return pid_blurs
    else:
        # if not there, then they don't have any blurs
        return 0

def get_relative_perf(x):
    # calculate their performance relative to what is expected based on random guessing
    rand_perf = random_perf[random_perf.pid == x].random_test.iloc[0]
    actual_earnings = random_perf[random_perf.pid == x].actual_earnings.iloc[0]
    rel_perf = (actual_earnings*0.01) - (rand_perf*0.01)
    
    return rel_perf

def aggregateTestData(data=None, exclude_pids=None, acc_reliable_only=False):
    
    pids = data.subjectID.unique()

    dOut = {'reliability':[],'value':[],'pOld':[],
            'pid':[],'bonus':[],'propResponse':[],
            'mean_rt':[],'n_rel':[],'n_unrel':[],'n_circle':[],'n_object':[],

            except:
                exclude_pids.append(pid)
                
        bonus = test_data.bonus.mean()

def gen_random_performance(data=None,n_sims=None):

    # Let's try and model guessing behavior
    # e.g. how much would we expect them to earn if they were just guessing?
    import random

    test_data = data[(data.test_part == 'test')]

    random_perf = {'pid':[],'random_test':[],'actual_earnings':[]}
    for pid in test_data.subjectID.unique():

        pid_data = test_data[test_data.subjectID == pid]

        earnings = pid_data[pid_data.earned != 'none'].earned
        total_earnings = np.sum(pd.to_numeric(earnings))

        sim_earnings = []
        for sim in range(n_sims):

            test_earnings = 0

            for i_row, row in pid_data.iterrows():

                reliableSample = np.round_(np.random.normal(0.6,0.05,1),decimals=2)
                unreliableSample = np.round_(np.random.normal(0.6,0.2,1),decimals=2)

                old_val = row.old_value
                deck_type = row.deck_type

                if old_val == 'none':
                    obj_val = random.choice([0,1.2])
                elif old_val == '0.00':
                    obj_val = 0
                elif old_val == '1.20':
                    obj_val = 1.2

                if deck_type == 'blue':
                    deck_val = reliableSample[0]
                else:
                    deck_val = unreliableSample[0]

                test_earnings+=random.choice([deck_val,obj_val])

            sim_earnings.append(test_earnings)

        random_perf['pid'].append(pid)
        random_perf['random_test'].append(np.mean(sim_earnings))
        random_perf['actual_earnings'].append(total_earnings)

    random_perf = pd.DataFrame.from_dict(random_perf)

    return random_perf

def get_best_strategy(strategyDf):

        pid_accuracy = agg_data[(agg_data.pid == pid)] \

        unrel = np.array(agg_data[(agg_data.reliability == "unreliable") & \
                                (agg_data.value == value)][choice_type])
        return rel - unrel
        
    highOldChosenDiff = get_choice_diff(value="high",choice_type="n_old_chosen")
    lowOldChosenDiff = get_choice_diff(value="low",choice_type="n_old_chosen")
    newChosenDiff = get_choice_diff(value="high",choice_type="n_new_noprop")

    reliabilityDiff = np.array(agg_data[agg_data.reliability == "reliable"].n_rel_noprop) - \
                      np.array(agg_data[agg_data.reliability == "unreliable"].n_unrel_noprop)
    reliabilityDiff = reliabilityDiff[:len(reliabilityDiff):2]
    pidList = pidList[:len(pidList):2]
    
    chosenDiff = [i for j in [lowOldChosenDiff,highOldChosenDiff,newChosenDiff] for i in j]
    pidList = [i for j in [pidList,pidList,pidList] for i in j]
    reliabilityDiff = [i for j in [reliabilityDiff, reliabilityDiff,reliabilityDiff] for i in j]
    value = [i for j in [["low"] * len(lowOldChosenDiff),
                         ["high"] * len(highOldChosenDiff),
                         ["new"] * len(newChosenDiff)] for i in j]

    relDiffDf = {'pid':pidList,
                 'rel_chosen - unrel_chosen':chosenDiff,
                 'n_rel - n_unrel':reliabilityDiff,
                 'value':value}

    relDiffDf = pd.DataFrame.from_dict(relDiffDf)
    
    return relDiffDf
